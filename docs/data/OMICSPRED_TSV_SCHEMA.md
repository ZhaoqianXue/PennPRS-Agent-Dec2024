# OmicsPred TSV Data Schema Documentation (Ultimate Version)

This document describes the schema of the `omicspred_scores_full.tsv` file. This version incorporates deep metadata from both the Score-level and Dataset-level APIs.

## File Information
- **Format**: Tab-Separated Values (TSV)
- **Primary Key**: `id`
- **Total Rows**: ~304,391

## Column Definitions

### 1. Core Identifiers & Links
| # | Column Name | Description |
| :--- | :--- | :--- |
| 1 | `id` | Global unique identifier for the score. |
| 2 | `score_web_url` | Direct link to the OmicsPred web page for this score. |
| 3 | `name` | Specific name for the score model. |
| 4 | `trait_reported` | Reported name of the protein/metabolite. |
| 5 | `trait_reported_id` | External identifier (e.g., UniProt ID). |

### 2. Model & Platform
| # | Column Name | Description | Note |
| :--- | :--- | :--- | :--- |
| 6 | `method_name` | Statistical training method. | - |
| 7 | `method_params` | Method-specific parameters. | - |
| 8 | `variants_number` | **Number of SNPs** in the model. | - |
| 9 | `variants_genomebuild`| Genome build (e.g., GRCh37). | - |
| 10 | `dataset_id` | ID of the study (e.g., OPD000001). | - |

### 3. Study Design (Dataset Level)
| # | Column Name | Description |
| :--- | :--- | :--- |
| 11 | `study_train_n` | **Total Sample Size (N)** used for training the model. |
| 12 | `study_train_cohorts` | Names of cohorts used in training (e.g., INTERVAL). |
| 13 | `study_train_ancestries`| Ancestries used in training. |
| 14 | `study_valid_n` | **Total Sample Size (N)** used for independent validation. |
| 15 | `study_valid_cohorts` | Names of cohorts used in validation. |
| 16 | `study_valid_ancestries`| Ancestries used in validation. |
| 17 | `study_omics_type` | Category of data (protein, metabolite, etc.). |

### 4. Hardware & Tissue
| # | Column Name | Description |
| :--- | :--- | :--- |
| 18 | `platform_name` | Instrument/Assay provider. |
| 19 | `platform_full_name` | Full descriptive name of the platform. |
| 20 | `platform_technic` | Technology type. |
| 21 | `tissue_label` | Sample source (e.g., blood plasma). |

### 5. Biological Context (Omics)
| # | Column Name | Description | Note |
| :--- | :--- | :--- | :--- |
| 22 | `genes_names` | Symbols of associated genes. | Separated by `|` |
| 23 | `genes_external_ids` | Ensembl IDs. | Separated by `|` |
| 24 | `proteins_names` | Full name of the protein. | - |
| 25 | `proteins_external_ids` | UniProt IDs. | - |

### 6. Publication & Data Access
| # | Column Name | Description |
| :--- | :--- | :--- |
| 26 | `pub_title` | Research paper title. |
| 27 | `pub_journal` | Journal name. |
| 28 | `pub_doi` | DOI link. |
| 29 | `pub_pmid` | PubMed ID. |
| 30 | `pub_date` | Publication date. |
| 31 | `url_gwas_sumstats` | URL for raw GWAS summary statistics. |
| 32 | `url_scoring_files` | URL for genetic score weights. |
| 33 | `url_validation_results`| URL for cohort-specific validation metrics. |
| 34 | `license` | Usage license (CC BY 4.0). |

---

## Technical Notes
- **Empty Values**: Represented by an empty string.
- **Lists**: Multi-value fields (like cohorts or synonyms) use `|` as a logical OR separator.
- **Updates**: This schema is generated by `download_omicspred.py` and matches the latest API structures.
